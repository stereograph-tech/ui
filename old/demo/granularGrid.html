Granular Grid Control

Introduction

Meet the .fontSize{N} class. The {N} represents the various font sizes (in px) that you can use. By default, the available sizes range from 5 - 30. For example: 


    <div class='fontSize12'> 
        <!-- 
            This is an object somewhere in the layout.
            Now all of its children inherit 12px as 
            the value for their EM,
            so, 12px = 1em.
        --> 
    </div>

How it's applied

Generally, you will only need to apply the CSS class to the element to see immediate results. However, the whole point of this framework is to provide solid, responsive functionality. For example, a UX designer might desire to increase the size of an object only at a certain device dimension or orientation, such as landscape vs. portrait. For that, we need to take a look at a more advanced example.

Cross-dimensional granular control

This is where Framework shines. In order to achieve various results on various dimensions, we can use a variety of @media queries to control the sizes of individual objects across the app.

Examples

Body font-size: 10px, Numpad font-size: 5px

Body font-size: 10px, Numpad font-size: 8px

Body font-size: 10px, Numpad font-size: 10px

Body font-size: 10px, Numpad font-size: 12px

Body font-size: 10px, Numpad font-size: 15px

Note: When you add the .fontSize{N} class on any given UI object, the object and its contents will no longer align to the body grid size. Rest assured, that the object's internal grid is still functioning normally!
