Portrait & Landscape

Introduction

The overarching concept when it comes to switching elegantly between portrait and landscape is to consider some fundamental principles:

  1. Don't redesign the wheel; there is no need to redesign the entire app in the event that someone rotates their device. Users shouldn't have to relearn the entire UI when the orientation changes.
  2. A horizontal landscape layout should switch to a vertical portrait layout, and vice versa.
  3. Listen to KISS: Keep It Simple, Stupid.

How it's applied

numpad.css: 


    /* any screen width below 1024px */
    @media (max-width: 1024px) {
        .numpad {
            font-size: 8px;
            min-width: 33em;
            width: 100%;
        }
        #numpadContainer {
            flex: 1 0 auto;
            width: calc(100% - 6em);
        }
    }

Important considerations

One problem with using static CSS classes in HTML (and remember, no JavaScript), is that it becomes a challenge to change individual widths, heights, margins, and paddings of various elements. In the next set of cases, we'll take a look at ways to use ghost elements, @media queries, and other techniques to keep things simple and intuitive.
