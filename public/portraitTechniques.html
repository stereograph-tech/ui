Portrait Techniques

Introduction

The overarching concept when it comes to switching elegantly between portrait and landscape is to consider some fundamental principles:

  1. Don't redesign the wheel; there is no need to redesign the entire app in the event that someone rotates their device. Users shouldn't have to relearn the entire UI when the orientation changes.
  2. A horizontal landscape layout should switch to a vertical portrait layout, and vice versa.
  3. Listen to KISS: Keep It Simple, Stupid.

How it's applied

numpad.css: 


    <div class='col'> Column 1 </div>
    
    <!-- 
        This "shim" column will collapse under 1024px.
        Normally, it will display the common 4GU (grid unit)
        gutter.
    -->
    <div class='col width4 height4'> Column 2 </div>
    
    <div class='col'> Column 1 </div>

By changing the flow of a row-based layout to a column-based layout when switching to portrait view, the height of the shim column above will become the vertical spacer between columns. 


    /* Layout.css */
    /* screen width under 1024px (portrait) */
    @media (max-width: 1024px) {
	
        section[role=main].panoramic, section[role=main].row {
            flex-direction: column;
        }
    }

Some considerations

The UX designers have been careful to keep the gutter between columns consistent throughout the layout. A hearty 40px (4GU) margin has been the chosen spacing, which means we can likely create a simple CSS class called "gutter" to maintain our spacing. In case this might change in different orientations, it will be easy enough to add a few media queries to control gutter spacing, if necessary.
