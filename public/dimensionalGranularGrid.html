Dimensional Granular Control

Introduction

Sometimes we can't escape the need to write extra CSS. You simply can't pit a generic grid framework against a complex UX with unique requirements and expect things to "just work." But the same core principle of using the EM to control overall individual object scaling still applies.

What it does

It's a little hard to see in these images. What you're looking at is the numpad scaling in relation to the rest of the page contents, given the current device dimensions. Note that while the list also looks like it's scaling from image to image, don't be fooled. Nothing except for the numpad is scaling in the images.

How it's applied

This is where control-specific CSS comes into play...

numpad.css:


    .numpad {
        /* default font-size = 10px inherited from the body */
    }

    /* any screen width below 1024px */
    @media (max-width: 1024px) {
        .numpad {
            font-size: 8px;
        }
    }

    /* any screen width above 1600px */
    @media (min-width: 1600px) {
        .numpad {
            font-size: 15px;
        }
    }

What about device orientation?

This leads us further down the rabbit hole. Device widths may change, but on devices that support portrait and landscape orientations, we enter an interesting new realm of unique challenges and a shift in gravity.

Note: changing the font size of an individual object in the body will cause the object and its children to no longer align to the body's grid. However, the object and its contents are still aligned to their own internal grid. Each grid unit of that new internal grid is equal to the object's specified font-size.
